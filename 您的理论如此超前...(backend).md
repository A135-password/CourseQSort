# 后端亮点

## 55-bit 位图冲突检测

学生端使用 **55 位二进制位图**（5 天 × 11 节/天）实现毫秒级冲突检测：

```
build_bitmap([(day, period), ...]) → "0x0F00..."
has_conflict(bitmap_a, bitmap_b)   → True/False
```

前端拿到待选课程位图后，与本地位图做一次按位与即可判断冲突，无需请求后端，选课高峰期也能支撑 5000+ 并发。

## 遗传算法排课引擎

完整的遗传算法实现，支持：

| 组件     | 方案                                                            |
| -------- | --------------------------------------------------------------- |
| 编码     | `(course_id, day, period, teacher_id, classroom_id)` 元组列表 |
| 初始化   | 随机分配时段 + 按容量智能匹配教室                               |
| 选择     | 锦标赛选择                                                      |
| 交叉     | 均匀交叉                                                        |
| 变异     | 时段/教室/教师独立变异                                          |
| 精英保留 | 前 5% 直接进入下一代                                            |
| 收敛判断 | 适应度 ≥ 0.999 或 50 代无改进                                  |

### 硬约束

- 教师冲突：同一教师同一时段只能排一门课
- 教室冲突：同一教室同一时段只能排一门课
- 教室容量：`expected_student_count ≤ classroom.capacity`
- 教师禁排：`Teacher.unavailable_slots` 指定的时段不排课
- 午休禁排：第 5 节不排课
- 体育课后禁排：体育课后至少空 1 节

### 软约束

- **日课时方差最小化**：避免学生某天满课、某天空闲的极端情况，方差越小适应度越高
- **辅修时段保护**：占用受保护时段的课程越多，惩罚越重，促使专业课绕开辅修热门时段

## 多策略调度优化器

`optimizer.py` 根据数据情况自动选择策略：

```
有 CourseScheduleItem 源数据？ → 按周次展开直接导入（保留原始排课结构）
无源数据？                    → 遗传算法从零生成
```

## 算法参数热配置

所有遗传算法参数（种群大小、变异率、交叉率、权重等）存储在数据库 `algorithm_config` 表中，教务通过 `PUT /api/v1/admin/algorithm-config/` 接口调整，**无需重启服务、无需修改代码**，下次排课任务自动加载最新配置。

## 冲突预分析（课程碰撞检测）

对指定学期的所有课程做 O(n²) 配对扫描，基于 `CourseScheduleItem` 的 day_of_week × period 重叠判断时间冲突，返回冲突课程对及冲突人数/比率，供教务提前发现大面积"撞车"风险。

## JSON 数据导入管道

支持多种来源系统的 JSON 格式兼容：

| 格式                                   | 说明          |
| -------------------------------------- | ------------- |
| 平铺数组`[...]`                      | 直接解析      |
| `{"total": N, "rows": [...]}`        | 单对象包 rows |
| `[{"total": N, "rows": [...]}, ...]` | 分页数组      |

自动处理 `teachingTimePlaceStr` 的 4/5/6 段变体（有无地点、有无教师），校区前缀剥离，教室地点与实验室识别。

## Django 最佳实践

- **Profile 扩展 User**：`OneToOneField` 扩展 role/major，不修改 auth_user 表
- **自定义权限**：`IsAdminUser` / `IsStudentUser` 基于 profile.role 判断
- **JWT 无状态认证**：SimpleJWT，access 30 分钟 + refresh 7 天 + 黑名单登出
- **异步任务就绪**：Celery + Redis 配置已预置，`tasks.py` 可直接切换为异步执行
- **SQLite/MySQL 双适配**：开发用 SQLite 零配置，生产设置环境变量即可切换 MySQL

## 完整的数据生命周期

```
爬虫 JSON → import_from_crawled_json (Management Command)
  ↓
Course / Teacher / Classroom / Major / CourseScheduleItem
  ↓
教务 CRUD API → 调整课程/教师/教室
  ↓
算法参数配置 + 辅修时段保护
  ↓
生成排课方案 → 遗传算法 → ScheduleEntry
  ↓
评估 → 发布 → 学生端可见
  ↓
学生选课/退课 + 位图冲突检测
```
